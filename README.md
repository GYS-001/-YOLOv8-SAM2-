# 一、项目概述
本项目为**机器视觉方向工业级落地项目**，基于 YOLOv8 目标检测与 SAM2 实例分割算法，实现**离线视频文件 + USB实时摄像头**双模式车辆跟踪系统。
项目核心目标：**在低显存消费级硬件上，解决SAM2原生模型无法实时、显存泄漏、高分辨率推理卡顿等工业痛点**，完成端到端视觉感知系统的稳定部署。

# 二、核心技术栈
| 模块 | 技术选型 |
| ---- | ---- |
| 编程语言 | Python 3.10 |
| 深度学习框架 | PyTorch 2.0+ |
| 并行计算 | CUDA 12.4、TF32 张量加速 |
| 视觉算法 | YOLOv8s（目标检测）、SAM2-tiny（实例分割） |
| 图像处理 | OpenCV、NumPy |
| 精度模式 | BF16 混合精度推理 |
| 部署模式 | 单帧实时推理、无状态边缘部署 |

# 三、开发背景与需求
1. **场景需求**：适配安防监控4K俯拍场景，实现车辆检测、像素级分割、实时跟踪
2. **硬件约束**：仅8G显存GPU，禁止显存溢出、禁止内存泄漏
3. **性能需求**：实时摄像头≥20FPS，支持无限时长视频稳定运行
4. **功能需求**：离线视频/摄像头双模式一键切换，无报错稳定部署

# 四、全流程开发步骤（完整无删减）
## 4.1 基础环境搭建与底层编译适配
项目初期完成**Windows平台CUDA扩展全流程编译**，配置SAM2依赖环境、动态链接库路径，解决原生框架在Windows系统下的算子编译失败、依赖冲突、CUDA版本不兼容等基础问题，为后续模型推理奠定稳定环境。

## 4.2 算法基线搭建与功能验证
基于 YOLOv8 完成车辆类别的专属检测配置，结合 SAM2 完成检测框驱动的实例分割；实现基础的车辆检测、掩码分割、可视化绘制功能，完成算法基线的功能闭环验证。

## 4.3 核心架构重构（项目核心突破）
摒弃 SAM2 原生 `SAM2VideoPredictor` 离线时序推理架构，**重构为 `SAM2ImagePredictor` 单帧实时推理架构**，彻底解决原生模型的长时记忆累积、显存随帧数线性增长、无法支持实时流的致命缺陷。

## 4.4 低显存硬件全链路优化
针对8G显存硬件约束，完成推理链路全维度优化：分辨率自适应推理、张量逐帧卸载、显存配额限制、混合精度适配，在不损失视觉效果的前提下，大幅降低算力与显存开销。

## 4.5 监控场景鲁棒性优化
针对俯拍监控小目标、树木遮挡、车辆重叠场景，基于NMS非极大值抑制优化检测逻辑，解决重复检测、空框、目标跟踪漂移等问题，提升系统在真实监控场景的稳定性。

## 4.6 双模式系统集成与部署
完成离线视频文件、USB实时摄像头双模式无缝集成，统一推理接口、统一可视化逻辑、统一错误处理机制，实现全系统零报错稳定部署。

# 五、核心技术难点与解决方案（含金量核心 | 完整踩坑记录）
本章节为项目**最高价值内容**，记录所有工业级痛点与原创解决方案，无任何浮夸描述。
## 5.1 难点1：Windows平台CUDA算子编译失败
**问题**：SAM2原生依赖Linux编译链，Windows下出现CUDA扩展编译报错、DLL动态链接库缺失
**解决方案**：手动配置CUDA系统路径、PyTorch依赖库路径，完成Windows专属编译适配，解决底层环境阻塞问题。

## 5.2 难点2：BF16混合精度推理数值异常
**问题**：启用BF16加速后，分割掩码出现精度丢失、数值溢出、画面撕裂
**解决方案**：完成精度适配调优，平衡张量加速与分割精度，在保留推理加速的前提下，彻底解决精度异常问题。

## 5.3 难点3：SAM2离线模型无法实时+显存泄漏
**问题**：原生VideoPredictor为离线批处理设计，显存随视频时长暴涨，8G显卡快速溢出，无实时能力
**解决方案**：**架构级重构**，替换为单帧无状态推理模式，显存占用恒定2-3G，支持无限时长推理，实现实时化改造。

## 5.4 难点4：4K高分辨率推理速度过慢
**问题**：3840×2160监控视频单帧推理耗时1s+，无法满足实时需求
**解决方案**：设计「高分辨率输入+低分辨率推理+坐标还原」方案，算力开销降低75%，视觉效果无损失，推理速度提升10倍。

## 5.5 难点5：监控场景目标漂移与空框误检
**问题**：俯拍场景下出现一车多框、车辆驶离后残留空框、跟踪框漂移
**解决方案**：强NMS去重、无效掩码过滤、最小目标尺寸约束，彻底解决鲁棒性问题。

## 5.6 难点6：模型选型适配消费级硬件
**问题**：SAM2大模型显存占用过高，8G显卡无法部署
**解决方案**：选型SAM2-tiny轻量化模型，完成监控场景精度验证，在满足分割需求的前提下，实现低硬件门槛部署。
